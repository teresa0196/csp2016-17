Git 사용하기

Git은 버전 관리 시스템의 종류입니다. 우리는 프로그램, 문서, 그리고 다른 정보의 형태의 변경 사항을 추적하기 위해 버전 관리 시스템을 사용합니다. 이 것은 우리가 작업하는데 오류 수정을 할 수 있게 해주고, 정보 기록을 보존할 수 있게 해줍니다. Git은 Github, Gitlab, BitBucket과 같은 공동 소프트웨어 개발 사이트를 만들기도 했습니다.

체크포인트 5: 변경사항 만들기

Git Bash에서 "usingGit" 레포지토리에 들어가신 후 git status를 입력하고 엔터를 누르세요. 변경사항이 없다는 메시지가 나올 것입니다. 이제 touch first.txt를 입력하고 엔터를 누르세요. 그리고 vim first.txt도 입력하신 후 엔터를 누르세요. 이제 다른 인터페이스가 보이게 될 것입니다. 이 것은 거의 모든 유닉스와 리눅스에 설치되어 있는 Vim 텍스트 편집기입니다. 혹은 Nano를 보실 수도 있습니다. 이 파일을 편집하기 위해선 키보드의 i를 눌러 삽입모드로 변경해야합니다.

이제 삽입모드에 있다면, Here is some text.를 파일에 입력하세요. 그리고 키보드의 Escape 키를 통해 정상 모드로 돌아가세요. 그리고 :wq를 입력하고 엔터를 눌러 변경사항을 저장하고 Vim을 종료하세요.

git status를 입력하고 엔터를 누르세요. 다음과 비슷한 내용을 보셔야합니다:

다음 단계에서 무엇을 해야 하는지 방법이 나올 것입니다. git add first.txt를 입력하고 엔터를 눌러 버전 관리 시스템에 파일을 추가하세요. git status를 이용해 변경 내용을 확인하세요.

다음으로 git commit -m "Add first.txt to version control"를 입력하고 엔터를 누르세요. 이 것은 현재 버전의 파일을 Git에 기록합니다. -m flag은 명령 다음에 메세지를 입력할 것이라는 것을 Git에게 알려줍니다. -m flag을 사용하지 않는다면 Vim 창이 나타나 메세지를 입력하도록 할 것입니다.

체크포인트 6: Branch 만들기

가끔 Git에서 "master branch"이라 불리는 마스터 버전을 방해하지 않고 프로젝트를 해야하는 경우가 있습니다. 그러기 위해선 간단히 git branch special을 입력하고 엔터를 누르세요. 그리고 git branch 를 눌러 레포지토리 안의 branch들을 확인하세요. 현재 branch는 초록색으로 나타납니다.

이제 git checkout special을 입력하고 엔터를 눌러 special branch로 이동하세요. 무엇이 변경되었는지 궁금하다면 git branch를 입력하세요.

또한 vim second.txt을 입력하신 후 엔터를 누르세요. 저번에 했던 것처럼 파일 내에 문자를 입력한 후 저장하세요. git add 와 git commit -m을 입력해 Git에 새 파일을 추가하실 수 있습니다. 그리고 git checkout master을 입력해 master branch로 돌아오세요.

이번엔 git merge special를 입력하고 엔터를 누르세요. "special" branch에서 생성한 모든 정보를 "master" branch로 전해줍니다. 이것을 형상화 해보자면, 길이 두 길로 나뉘었다가 다시 한 길로 합쳐지는 것과 같습니다. 정보들이 두 branch(또는 그 이상)로 나뉘었다가 다시 하나로 통합된 것입니다.

체크포인트 7: 삭제와 이름변경하기

이미 우리는 rm 과 mv를 이용해 삭제하고 이름을 변경하는 법을 알고 있습니다. Git안의 파일들을 작업하고 있을 때 그 도구들을 사용할 수 있지만, git rm and git mv를 사용해 과정들을 저장하는 것이 더 좋은 방법이라 할 수 있습니다.

touch deleteThisFile.txt renameThisFile.txt을 입력 후 엔터를 누르세요. 그리고 git add *를 통해 버전 관리에 기록되지 않은 모든 파일들을 추가하세요. 변경 사항을 저장하기 위해 git commit -m을 입력해야 합니다.

git rm deleteThisFile.txt을 입력하고 엔터를 누르세요. 이 명령은 디렉토리 내에서 파일을 삭제하고 Git에게도 변경사항을 전송합니다. git status를 사용해 이전 명령에 대한 결과를 확인해보세요. 파일을 삭제하지 않고 버전 관리에서만 지우고싶다면 git rm --cached를 입력하세요.

git commit -m을 사용해 변경사항을 저장하시길 바랍니다.

이번엔 git mv renameThisFile.txt third.txt을 입력하고 엔터를 누르세요. 이 명령은 파일 이름을 변경하고 Git에게 변경사항을 전해줍니다. git status를 통해 이전 명령에 대한 결과를 확인해보세요.

마지막으로 git commit -m을 통해 변경사항을 저장하세요.

체크포인트 8: gitignore

Git은 특정 파일, 파일 종류, 또는 디렉토리들을 무시할 수 있게 구성되어 있습니다. 이 것은 그룹 프로젝트를 진행할 때 다른 사용자에겐 도움이 되지 않는 노트를 보관할 때 도움이 됩니다. 본인은 필요하지만 다른 사용자들은 아닐 때, Git이 무시하도록 구성할 수 있습니다.

먼저, "usingGit"디렉토리 안에 "ignore.txt"라는 새 파일을 생성합니다. 그리고 vim .gitignore을 입력하신 후 엔터를 누르십시오. Vim 안에 있다면, i를 눌러 편집 모드에 들어가시고 ignore.txt를 입력하세요. 그리고 Vim을 저장하고 나가기 위해 Esc 키를 누른 후 :wq를 입력하고 엔터를 누르세요. git status를 입력하면 .gitignore 파일이 추가된 것만 보여야합니다. git add를 사용해 파일을 추적하고 commit하세요.

체크포인트 9: 담아두기

레포지토리에 변경사항을 만들었지만 아직 commit하지 않았다면 파일을 영구적으로 삭제하지 않고 변경사항과 변경된 파일들을 저장해 놓을 수 있습니다. 이를 통해 나중에 다른 작업을 수행한 후에도 나중에 이를 적용할 수 있습니다.

먼저 touch를 이용해 세 파일들을 생성하고 Git에 추가하세요. git status를 입력하면 미결정된 변경사항들을 보실 수 있습니다. git commit을 사용하는 대신, git stash save을 입력하세요. 파일들의 위치르 설명하는 메시지가 표시될 것입니다. 이제 ls를 사용해 새 파일들이 존재하지 않는지 확인해보세요.

체크포인트 10: 기록

특히 다른 사람들과 함께 작업중이라면 어떤 사람이 어떤 변경을 했는지 알아내는 것이 중요합니다. 이럴 때는 git log을 사용하세요.

git log을 입력하고 엔터를 눌러 무슨 일이 생기는지 확인하세요. 스페이스 바를 통해 스크롤 한 다음 q키를 눌러 종료할 수 있습니다.

git log --grep=를 입력해 검색 단어를 추가해보세요. Grep는 유닉스와 리눅스 시스템에서 사용되는 검색툴입니다. 만약 해당 단어와 일치하는 commit이 있다면, 엔터 키를 눌렀을 때 보여질 것입니다. 이제 한 번에 한 단어 이상을 검색하는 방법을 알아보세요. 그리고 git log --help을 입력하고 엔터를 눌러 당신의 브라우저에서 매뉴얼 페이지를 여세요. 레포지토리 내의 기록을 탐색하는데 필요한 최소 두 가지 옵션을 시도해 보세요.

체크포인트 11: 이런...

가끔 우리는 commit을 하기 전까지 실수를 깨닫지 못할 때가 있습니다. 당신은 두 옵션이 있습니다. commit기록에서 특정 commit으로 돌아가거나, 특정 commit으로 돌아간 후 모든 파일 변경사항을 삭제하는 방법이 있습니다.

몇 개의 텍스트 파일을 생성하고, 추가하고, commit하세요. 그리고 모든 새 파일들이 각자 다른 내용을 갖고 있도록 Vim을 통해 파일들을 편집하세요. 그리고 내용들을 commit하세요.

git log을 통해 commit 기록들을 불러내세요. 최근 commit들 중 두번째 commit(예시와는 다른 commit일 것입니다.)을 선택하고 commit ID를 강조표시하세요. 그리고 복사하세요.

이제 git reset을 입력하고 방금 전 복사한 commit ID를 붙여넣으세요. 이 명령은 마지막 commit을 실행 취소하지만 변경한 파일들은 저장할 수 있도록 합니다.

그리고 다시 git log을 통해 commit 기록들을 불러내세요. 최근 commit들 중 두번 째 commit을 강조표시 후 복사하세요.

git reset --hard을 입력하고 방금 전 복사했던 commit ID를 붙여넣으세요. 이 명령은 마지막 commit을 실행 취소하고 파일을 삭제합니다. 당신의 디렉토리는 체크포인트 10에 있던 것처럼 나타나야합니다.

체크포인트 12: 동기화

만약 당신이 원격 레포지토리에서 작업을 수행중이라면 다른 사용자들도 변경사항을 볼 수 있게 하기위해 동기화를 필수로 해야합니다. 동기화를 위해 가장 흔하게 쓰이는 방법은 git pull과 git push를 사용하는 것입니다.

"csp2016-17"디렉토리로 이동하세요. 그리고 git pull을 입력하세요. 당신의 레포지토리 복사본에 몇 변경사항들이 추가된 것이 보일 것입니다. 이제 코스 웹사이트 복사본과 동기화 된 것입니다. 윈도우 익스플로러의 디렉토리 또는 파인더에서 usingGit.html을 열어 이 레슨을 열어볼 수 있습니다.

이제 "usingGitPush"라는 하위 디렉토리로 이동해 텍스트파일을 생성하세요. 파일 이름은 yourName.txt이어야합니다(yourName은 실제 본인의 이름으로 대체하세요). 그리고 파일을 추가하고 commit한 다음 git push를 입력하고 엔터를 누르세요. 만약 현재 commit 전에 있다는 에러가 생긴다면 git pull --rebase를 입력하고 엔터를 누르세요. 마지막으로 git push를 입력하고 엔터를 누르세요.

체크포인트 13: 좋은 Git Commit

좋은 Git Commit을 위해선 일곱가지 규칙을 따라야합니다. Chris Beams가 작성한 일곱가지 규칙에 대한 설명글을 읽으세요.. 기본적으로 당신의 commit들은 다음과 같아야합니다:

  1. 제목과 문단은 한 줄 띄우세요
  2. 제목은 50자 이내로 적으세요
  3. 제목 줄은 대문자로 적으세요
  4. 제목은 마침표로 끝내지 마세요
  5. 제목은 명령조로 적으세요
  6. 문단은 72자 이내로 적으세요
  7. 문단으로 무엇을, 왜, 어떻게를 설명할 수 있도록 하세요

1, 6, 7 번은 오직 문단이 있는 commit 메세지를 작성할 때만 필요합니다. 대부분의 commit들은 문단을 작성할 필요가 없을 것입니다. "usingGit" 레포지토리 내에 일곱가지 규칙을 포함하는 텍스트 파일을 만든 후 추가하고 commit하세요. 당신의 commit 메세지가 규칙들을 정확히 따르는지 다시 확인해보세요.

손을 들고 올린다 선생님께 체크포인트를 확인 받으세요.

모든 체크포인트들에 서명을 받았다면 종이를 숙제 바구니에 제출하세요. 당신은 이제 프로젝트를 작업할 수 있습니다.

공부 팁

  • 일반적인 지침을 통해 특정 결과를 생성하는 법을 이해합니다.
  • 좋은 Git commit에 대한 일곱가지 규칙을 알고 그에 따른 좋은 Git commit들을 만들어냅니다.
  • Bash나 PowerShell 명령들을 사용하는 해결책이 있더라도 Git으로 해결하는 최상의 방법을 이해합니다.
